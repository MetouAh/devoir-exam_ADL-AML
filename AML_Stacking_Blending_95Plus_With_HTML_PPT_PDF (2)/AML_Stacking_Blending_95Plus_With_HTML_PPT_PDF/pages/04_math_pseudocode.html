Fondements mathématiques et pseudo-code

Notation complète : dataset, modèles de base, meta-learner, matrice Z, extension probabiliste et risque empirique.

1. Notation

Dataset
D = {(x_i, y_i)} pour i = 1,...,n
Modèles
h_1, h_2, ..., h_M : modèles de base g : meta-learner K : nombre de folds

2. Construction OOF

Pour chaque modèle de base h_j et chaque fold k, on entraîne h_j sur toutes les données sauf D_k, puis on prédit uniquement D_k.

Modèle entraîné sans le fold k
h_j^(-k) entraîné sur D \ D_k
Méta-représentation
Z[i,j] = h_j^(-k)(x_i), pour x_i ∈ D_k

3. Matrice des meta-features

Sorties scalaires
Z ∈ R^(n × M)
Sorties probabilistes multi-classes
Z[i,j,c] = P_j(y = c | x_i) Z ∈ R^(n × M × C)

4. Apprentissage du meta-learner

Risque empirique régularisé
g* = argmin_g Σ_i L(y_i, g(Z_i)) + λ Ω(g)

La régularisation Ω(g) empêche le meta-learner de sur-apprendre les erreurs des base learners. En pratique, une régression logistique régularisée est souvent un excellent choix initial.

5. Pseudo-code complet

Entrée : dataset D, base learners h_1...h_M, meta-learner g, K folds

Initialiser Z ∈ R^(n × M)

Pour chaque modèle h_j :
    Pour chaque fold k :
        entraîner h_j^(-k) sur D \ D_k
        prédire les exemples de D_k
        stocker les prédictions dans Z[:, j]

Entraîner le meta-learner g sur Z

Réentraîner chaque h_j sur tout D_train

Pour un nouvel exemple x* :
    z* = [h_1(x*), h_2(x*), ..., h_M(x*)]
    retourner ŷ* = g(z*)

Approfondissement mathématique — version soutenance

Cette section renforce le contenu mathématique minimal : elle explique non seulement les formules, mais aussi leur rôle statistique dans la prévention de l'overfitting du meta-learner.

1. Données, base learners et folds
D = {(x_i, y_i)}_{i=1}^{n} H = {h_1, h_2, ..., h_M} K folds : D = D_1 ∪ D_2 ∪ ... ∪ D_K
2. Construction Out-of-Fold exacte
Pour chaque modèle h_j et chaque fold k : h_j^{(-k)} = train(h_j, D \ D_k) Pour chaque x_i ∈ D_k : Z[i,j] = h_j^{(-k)}(x_i)
3. Cas probabiliste multi-classes
Z[i,j,c] = P_j(y=c | x_i) Z ∈ R^{n × M × C} Flatten(Z_i) = [P_1(c_1|x_i), ..., P_M(c_C|x_i)]
4. Risque empirique régularisé
g* = argmin_g Σ_i L(y_i, g(Z_i)) + λ Ω(g) L : cross-entropy ou log-loss Ω(g) : pénalisation L2, profondeur limitée, shrinkage λ : contrôle de complexité du meta-learner
Pourquoi la fuite est grave ? Si Z est construit avec des prédictions in-sample, alors Z[i,j] ≈ y_i pour certains modèles très flexibles. Le meta-learner apprend alors la capacité du modèle à mémoriser le train set, pas sa capacité à généraliser. Le score test devient instable et trop optimiste.
SituationEffet statistiqueSolution
Prédictions in-sampleBiais optimiste, overfitting du niveau 1OOF strict
Base learners très corrélésGain faible par rapport au Best SingleDiversifier les familles de modèles
Meta-learner trop complexeVariance élevée sur ZLogistic Regression régularisée comme baseline
Probabilités non calibréesSoft voting et stacking probabiliste biaisésCalibratedClassifierCV ou calibration post-hoc
