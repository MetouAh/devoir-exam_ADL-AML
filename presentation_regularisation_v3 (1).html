Partie 1Sur-apprentissage dans les réseaux profonds

Le problème : sur-apprentissage

Pourquoi les réseaux profonds mémorisent-ils au lieu d'apprendre ?

Définition

Un modèle est en sur-apprentissage quand il mémorise les exemples d'entraînement — y compris le bruit — sans capturer la structure générale. Il excelle sur les données vues, mais échoue sur les nouvelles.

Décomposition Biais–Variance
E[(y − ŷ)²] = Biais² + Variance + σ²
Biais²Erreur systématique — modèle trop simple. Sous-apprentissage.
VarianceSensibilité au bruit — réseau profond non régularisé. Sur-apprentissage.
σ²Bruit irréductible — inhérent aux données.
Symptômes
  • Erreur train → 0 % mais erreur de test reste élevée
  • Grand écart entre courbe train et courbe validation
  • Le modèle est trop confiant sur des exemples ambigus
PythonDiagnostic de surapprentissage
import numpy as np

epochs = np.arange(1, 51)
train_loss = 2.0 * np.exp(-0.08 * epochs) + 0.05
val_loss   = 2.0 * np.exp(-0.05 * epochs) + 0.3 \
           + 0.008 * np.maximum(0, epochs - 20)

# Détection du sur-apprentissage : gap train/val croissant
best_epoch = np.argmin(val_loss) + 1
print(f"Early stop à l'epoch {best_epoch}")
print(f"Gap max : {(val_loss - train_loss).max():.3f}")
But de la régularisation : réduire la Variance sans augmenter le Biais.
Partie 2Dropout — Principe et intuition

Dropout — Principe

Srivastava, Hinton, Krizhevsky, Sutskever & Salakhutdinov — JMLR 2014

Idée centrale

À chaque forward pass pendant l'entraînement, chaque neurone est désactivé aléatoirement avec une probabilité p. Le réseau ne peut plus s'appuyer sur un neurone particulier — il est forcé d'apprendre des représentations redondantes et robustes.

Python · NumPyDropout from scratch
import numpy as np

def dropout(x, p=0.5, training=True):
    if not training:
        return x
    mask = (np.random.rand(*x.shape) > p).astype(np.float32)
    return x * mask / (1 - p)  # inverted dropout

x = np.array([1.0, 2.0, 3.0, 4.0])
print(dropout(x, p=0.5))
# Ex : [0. 4. 6. 0.]  — neurones nuls ou doublés
Valeurs typiques de p
  • Couches fully-connected : p = 0.4 à 0.5
  • Couches convolutives : p = 0.1 à 0.25
  • Couche d'entrée : p = 0.1 à 0.2
  • p > 0.7 → sous-apprentissage
Partie 2Dropout — Formules mathématiques détaillées

Dropout — Formules et notations

Comprendre chaque symbole pas à pas

Espérance de la sortie (sans correction)
E[ŷ] = (1 − p) · y
E[ŷ]Espérance de la sortie masquée sur tous les tirages possibles de masques.
pTaux de Dropout — probabilité de désactivation d'un neurone.
yActivation originale sans masquage.
(1−p)Fraction d'actifs — facteur de réduction en espérance.
Inverted Dropout — méthode standard PyTorch
Entraînement : ãᵢ = mᵢ · aᵢ / (1 − p) Inférence : â = a ← pas de modification
ãᵢActivation masquée et rescalée — résultat final du neurone i.
mᵢMasque binaire Bernoulli(1−p) — 1=actif, 0=désactivé.
÷(1−p)Rescaling pour que E[ãᵢ] = aᵢ — train et test cohérents.
MC-Dropout — T passes à l'inférence
ŷ_MC = (1/T) · Σₜ f(x ; ωₜ)
TNombre de passes Monte-Carlo. T=50 suffit en pratique.
f(x;ωₜ)Sortie avec le masque aléatoire ωₜ — chaque tirage = un sous-réseau.
Python · NumPyMC-Dropout from scratch
import numpy as np

def mc_dropout(model_fn, x, p=0.5, T=50):
    # T passes stochastiques → moyenne + incertitude
    preds = [model_fn(x * (np.random.rand(*x.shape) > p) / (1-p))
             for _ in range(T)]
    preds = np.array(preds)
    return preds.mean(axis=0), preds.var(axis=0)

# mean → prédiction finale  |  var → incertitude épistémique
Avantages & Limites
  • Force des représentations redondantes et robustes
  • Augmente le temps d'entraînement (×2–3×)
  • Désactiver en inférence (model.eval())
  • Interaction complexe avec BatchNorm — placer Dropout après BN
Partie 3Batch Normalization — Motivation

Batch Normalization

Ioffe & Szegedy, ICML 2015 — Une révolution dans l'entraînement des réseaux profonds

Le problème : Internal Covariate Shift

Lors de l'entraînement, la mise à jour des poids d'une couche L modifie la distribution des activations vue par la couche L+1. Cette couche doit s'adapter en permanence à une cible mouvante — ralentit l'entraînement et exige de petits taux d'apprentissage.

Pipeline BatchNorm dans un bloc
1
Couche Linéaire / Conv : z = Wx + b
2
BatchNorm : normalise z → x̂ (μ=0, σ²=1)
3
Scale & Shift : y = γ · x̂ + β (appris)
4
Activation : ReLU(y)
Bénéfices : taux d'apprentissage ×10 à ×100 plus grands · convergence 2–14× plus rapide · légère régularisation par le bruit du batch.
Python · NumPyBatchNorm forward from scratch
import numpy as np

def batchnorm(x, gamma, beta, eps=1e-5):
    mu  = x.mean(axis=0)          # moyenne sur le batch
    var = x.var(axis=0)           # variance sur le batch
    x_hat = (x - mu) / np.sqrt(var + eps)
    return gamma * x_hat + beta    # scale & shift appris

x     = np.random.randn(4, 3)   # batch=4, features=3
gamma = np.ones(3); beta = np.zeros(3)
y = batchnorm(x, gamma, beta)
print("μ ≈ 0 :", y.mean(axis=0).round(4))  # [0. 0. 0.]
print("σ²≈ 1 :", y.var(axis=0).round(4))   # [1. 1. 1.]
Partie 3Batch Normalization — Formules complètes

BatchNorm — formules et notations

Mini-batch B = {x₁, x₂, …, xₘ} — calcul séparé par feature

Étape 1 — Moyenne du mini-batch
μ_B = (1/m) · Σᵢ xᵢ
μ_BMoyenne empirique sur les m exemples du batch, par feature.
mTaille du batch — 32, 64 ou 128 typiquement.
xᵢActivation du i-ème exemple pour la feature courante.
Étape 2 — Variance du mini-batch
σ²_B = (1/m) · Σᵢ (xᵢ − μ_B)²
σ²_BVariance empirique — dispersion des activations autour de leur moyenne.
(xᵢ−μ_B)²Carré de l'écart — toujours positif.
Étape 3 — Normalisation (centrer–réduire)
x̂ᵢ = (xᵢ − μ_B) / √(σ²_B + ε)
x̂ᵢActivation normalisée — distribution N(0,1) en espérance.
εConstante numérique ≈ 1e-5 — évite la division par zéro.
Étape 4 — Scale & Shift (paramètres appris)
yᵢ = γ · x̂ᵢ + β
γScale appris — initialisé à 1. Module l'amplitude.
βShift appris — initialisé à 0. Décale la moyenne. Remplace le biais de la couche.
yᵢSortie finale — distribution apprennable, pas forcément N(0,1).
À l'inférence : statistiques globales par EMA

Pendant l'entraînement on maintient : μ_global ← α·μ_global + (1−α)·μ_B. À l'inférence, ces valeurs fixes remplacent les stats du batch — sinon un batch de taille 1 serait normalisé à 0 et perdrait toute information.

Partie 4Layer Normalization — Formule et comparaison

Layer Normalization

Ba, Kiros & Hinton, 2016 — Conçu pour les RNN et Transformers

Motivation

BatchNorm dépend de la taille du batch. Pour les Transformers, RNN, séquences variables, il fallait une normalisation indépendante du batch. LayerNorm normalise sur les features de chaque exemple séparément.

Formule LayerNorm — pour l'exemple i
μ_l = (1/H) · Σₕ aₕ σ²_l = (1/H) · Σₕ (aₕ − μ_l)² x̂ₕ = (aₕ − μ_l) / √(σ²_l + ε) yₕ = γ · x̂ₕ + β
HNombre de features — LayerNorm moyenne sur les H features d'UN exemple.
aₕActivation de la h-ème feature pour l'exemple courant.
μ_l, σ²_lMoyenne et variance par exemple — différent de BatchNorm qui utilise le batch.
Python · NumPyLayerNorm from scratch
import numpy as np

def layernorm(x, gamma, beta, eps=1e-5):
    # axis=1 : moyenne/var sur les features de CHAQUE exemple
    mu  = x.mean(axis=1, keepdims=True)
    var = x.var(axis=1, keepdims=True)
    return gamma * (x - mu) / np.sqrt(var + eps) + beta

x     = np.random.randn(4, 3)   # batch=4, H=3
gamma = np.ones(3); beta = np.zeros(3)
y = layernorm(x, gamma, beta)
print("μ par exemple :", y.mean(axis=1).round(4))  # ≈ 0
Règle pratique : CNN/vision (grand batch) → BatchNorm · Transformer/NLP/batch variable → LayerNorm · LLM modernes (LLaMA, Mistral) → RMSNorm.

Partie 5Label Smoothing — Régularisation des cibles

Label Smoothing

Szegedy et al., CVPR 2016 · Müller, Kornblith & Hinton, NeurIPS 2019

Problème : surconfiance avec les cibles one-hot

Avec des cibles one-hot dures (0 ou 1), le réseau cherche à saturer le softmax — logits → ±∞. Il devient surconfiant : 100% pour la bonne classe, 0% pour les autres. Nuit à la calibration et à la généralisation.

Formule de la cible lissée
ỹ_k = (1 − α) · y_k + α/K
ỹ_kCible lissée pour la classe k — remplace la cible one-hot dans la perte.
αTaux de lissage — fraction redistribuée uniformément. Standard : α = 0.1.
y_kCible one-hot — 1 pour la bonne classe, 0 pour les autres.
KNombre de classes — α/K = probabilité minimale garantie pour chaque classe.
Exemple numérique (α=0.1, K=10) :
Bonne classe → (1−0.1)·1 + 0.1/10 = 0.91
Mauvaises classes → (1−0.1)·0 + 0.1/10 = 0.01 chacune
Python · NumPyLabel Smoothing from scratch
import numpy as np

def label_smooth(y_onehot, alpha=0.1):
    K = y_onehot.shape[1]
    return (1 - alpha) * y_onehot + alpha / K

# Exemple : 4 classes, alpha=0.1
y   = np.eye(4)[[0]]           # one-hot [1, 0, 0, 0]
y_s = label_smooth(y)
print("one-hot :", y[0])        # [1.  0.  0.  0. ]
print("lissé   :", y_s[0])     # [0.925 0.025 0.025 0.025]
Partie 6Data Augmentation comme régularisation implicite

Data Augmentation

Régularisation implicite par diversification artificielle des données

Principe

En appliquant des transformations préservant la sémantique, on augmente la diversité effective du dataset et on force le modèle à être invariant à ces transformations — sans collecter de nouvelles données.

Mixup — Zhang et al., ICLR 2018
x̃ = λ·xᵢ + (1−λ)·xⱼ ỹ = λ·yᵢ + (1−λ)·yⱼ λ ~ Beta(α, α)
x̃, ỹExemple synthétique — interpolation linéaire de deux exemples réels.
λPoids tiré de Beta(α,α). Les images ET les étiquettes sont interpolées.
Python · NumPyMixup from scratch
import numpy as np

def mixup(x, y, alpha=0.4):
    lam = np.random.beta(alpha, alpha)
    idx = np.random.permutation(len(x))
    return lam*x + (1-lam)*x[idx], lam*y + (1-lam)*y[idx]

# batch de 8 images 32×32 RGB, 10 classes
x = np.random.randn(8, 32, 32, 3)
y = np.eye(10)[np.random.randint(0, 10, 8)]
x_mix, y_mix = mixup(x, y)
print("label mixé :", y_mix[0].round(2))
Partie 7Weight Decay — Régularisation L1 / L2

Weight Decay

Principe de parcimonie — préférer les modèles avec des poids petits

Principe

On ajoute à la fonction de perte un terme de pénalité sur les poids. Cela incite le modèle à utiliser des poids faibles — il ne peut pas s'appuyer sur des features très spécifiques aux données d'entraînement.

Régularisation L2 (Ridge / Weight Decay)
L_reg = L + λ · Σ wᵢ²
LPerte originale (cross-entropie, MSE…).
λCoefficient — typiquement 1e-4 à 1e-2. λ=0 : pas de régularisation.
Σwᵢ²Norme L2 des poids — pénalise les grands poids.
Régularisation L1 (Lasso)
L_reg = L + λ · Σ |wᵢ|
|wᵢ|Norme L1 — encourage la parcimonie : nombreux poids exactement nuls.
Python · NumPyWeight Decay (L2) from scratch
import numpy as np

def loss_l2(base_loss, weights, lam=1e-4):
    # Pénalise les grands poids → régularisation L2
    l2 = sum(np.sum(W**2) for W in weights)
    return base_loss + lam * l2

# Exemple
W1 = np.random.randn(4, 4)
W2 = np.random.randn(4, 2)
loss = loss_l2(0.8, [W1, W2])
print(f"Loss régularisée : {loss:.4f}")
# PyTorch : Adam(params, weight_decay=1e-4)
L1 vs L2
  • L2 : standard DL · Adam weight_decay=1e-4 · différentiable partout
  • L1 : parcimonie (feature selection) · non différentiable en 0
  • Elastic Net : combinaison L1+L2 — compromis

ConclusionSynthèse et ouverture

Conclusion & Synthèse

Ce qu'il faut retenir — et vers quoi ce domaine évolue

Points clés à retenir
  • Dropout : désactiver aléatoirement = ensemble bayésien implicite de 2ⁿ sous-réseaux. Réduit la variance.
  • BatchNorm : normaliser par mini-batch stabilise les distributions → convergence 2–14× plus rapide. Indispensable pour les CNN.
  • LayerNorm : normaliser par exemple, indépendamment du batch → standard des Transformers (GPT, BERT, LLaMA).
  • Label Smoothing : ramollir les cibles one-hot → meilleure calibration, modèle moins surconfiant.
  • Augmentation : diversifier les données = régularisation implicite la plus efficace sur petits datasets.
  • Lien Stacking : Dropout ≈ stacking bayésien implicite — même réduction de variance.
Tendances récentes
  • RMSNorm : LLaMA 3, Mistral — LayerNorm sans centrage, 15% plus rapide.
  • Stochastic Depth : Dropout sur des blocs entiers (EfficientNet v2).
  • Dropout adaptatif : taux variable selon la profondeur et la confiance.
Vers les chapitres suivants
  • LayerNorm → prérequis direct pour les Transformers
  • Label Smoothing → ancêtre des régularisations des VAE et diffusion models
  • MC-Dropout → approches d'incertitude bayésienne en DL
